# Simulation Code for the chapter "Differential Privacy for Free? Harnessing the Noise of Approximate Homomorphic Encryption"
## Code Structure
The code consists of six python files,
- main.py, which sets global parameters and contains the function calls which generate the graphs
- experiments.py, which contains plotting functions and privacy functions
- sensitivity.py, which calculates the sensitivity of Ridge Regression 
- variance.py, which calculates upper and lower bounds on the homomorphic noise variance
- message_dependence.py, which calcuates bounds on the message dependence
- plot_noise.py, which plots the distribution of the result when calculated homomorphically

and two C++ files,
- gen_data.cpp, which creates a regularised database
- noise.cpp, which runs the case study homomorphically and prints the result to a file

## Generating the Figures
Lines 27 - 31 of main.py contain the function calls which were used to generate the data, labelled with their respective figure numbers.
Figures can be regenerated by uncommenting the relevant line, and running
```
python3 main.py
```
The data will print in the terminal, formatted to be added in a tikzpicture environment in Latex. 
## Code Notation
Throughout the repo, we have used the same notation as the paper, and there is a notation key in Section 2.6. In message_dependence.py,
we additionally use the notation introduced for the sensitivity of the variance, namely
d_X = \left|\sigma^2_X - \sigma^2_{X'}\right|
$
where $X$ and $X'$ are adjacent databases.
Function outputs in variance.py are squared - i.e., `sigma_fresh` returns the variance of the fresh noise, not the standard deviation - and throughout the repo functions take variances as input. Similarly, the functions in message_dependence.py return $T^2$.
